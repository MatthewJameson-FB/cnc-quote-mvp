# 2026-04-29

- User prefers very low-token progress updates in the form: done / next / blocker / need-from-you.
- Heartbeat handling in this workspace is special-case: when explicitly instructed for heartbeat polls, reply exactly `HEARTBEAT_OK` only.
- Project focus today: broaden the prelead funnel from CNC-only toward "someone needs something made," especially 3D printing / make-this / replacement-part / STL-CAD intent, while still allowing CNC/fabrication leads.
- Do not change unless explicitly needed: cron route, Supabase schema, adapters structure, or admin UI.
- Keep hard rejects for supplier ads, jobs/careers, politics/news, tutorials/courses, and furniture/mattress/home decor; unknown location alone should not cause rejection.
- Current state reached: 3D-print and make-intent signals were expanded through scoring, qualification, and AI classification; production-safe logs for AI throughput are in place; a validated run produced materially better throughput and fresh inserts (latest noted run: sent_to_ai=25, ai_accepted=17, inserted=9), though quality tuning is still needed to reduce low-value hobby/consumer threads.
- Shared inbox setup decision: app-side email replies are routed via `reply_to` using `RESEND_REPLY_TO_EMAIL` with fallback to `RESEND_FROM_EMAIL`, but no inbound parsing, webhook, schema change, inbox UI, or mailbox provisioning was added; replies are handled manually in the monitored inbox.
- Quote email policy now includes a customer-facing reply footer and safe outbound email logs, while preserving the rule not to expose actual email addresses or secret values in logs.
- Added a lightweight `lib/photo-assessment.ts` helper for photo-based CAD requests. It evaluates photo readiness from uploaded photo URLs, measurements, description, material, and manufacturing type, but must never promise exact dimensions from photos alone or auto-generate CAD.
- Photo assessment integration is intentionally additive and non-breaking: results are stored in quote notes / internal surfaces only, used after quote submission for photo-based requests, shown in internal email/admin, and used to tailor checklist follow-up questions when more angles, scale reference, or a physical part is needed.
- Photo assessment rules to preserve: no measurements => `needs_scale_reference`; single photo => `needs_more_angles`; fit-critical terms like clip/thread/gear/tight fit/mating surface can force `needs_physical_part` or low confidence; always ask for a ruler/coin/reference object when scale is unclear.
- Latest validated state: shared-inbox reply handling and photo-assessment changes both passed `npm run lint` and `npm run build`.
